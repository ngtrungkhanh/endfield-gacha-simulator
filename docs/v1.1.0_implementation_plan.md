# Kế hoạch phát hành Endfield Gacha Simulator v1.1.0

> **Tài liệu lịch sử:** đây là kế hoạch ban đầu, không phải mô tả trạng thái runtime hiện tại. Bản triển khai sau cùng phát hành `A9EGacha.html` dạng single-file kèm SHA-256 thay cho ZIP, `dist/` chỉ chứa `index.html`, CSS và bundle JS, và bộ test hiện đã được mở rộng vượt quá 9 test ban đầu. Quy tắc gacha/chiến thuật hiện hành nằm trong [`gacha_rules.md`](gacha_rules.md) và [`strategies.md`](strategies.md); hướng dẫn build/phát hành hiện hành nằm trong [`README.md`](../README.md).

## 1. Mục tiêu và nguyên tắc

Phát hành `v1.1.0` dưới dạng website song ngữ Việt/Anh có thể tải về, giải nén và mở trực tiếp `index.html` mà không cần Node.js hoặc local server. Phiên bản này đồng thời hoàn thiện bố cục desktop/mobile, khả năng sử dụng bằng bàn phím, quy trình build/package và kiểm thử offline.

Các tài liệu [`gacha_rules.md`](gacha_rules.md), [`strategies.md`](strategies.md), [`bookkeeping.md`](../data/bookkeeping.md) và [`detailed_gacha_run.md`](../reports/detailed_gacha_run.md) tiếp tục là nguồn sự thật. Không thay đổi luật gacha, chiến thuật, xác suất hoặc dữ liệu bookkeeping nếu không có test tái hiện bug và bằng chứng đối chiếu tài liệu.

Trạng thái repository đã rà soát tại commit `5d9f4eb`:

- Giao diện tĩnh nằm trong `index.html`, CSS trong `css/style.css`, JavaScript được bundle từ `js/app.js` sang `js/bundle.js` bằng esbuild.
- Runtime hiện còn ba phụ thuộc mạng: Google Fonts preconnect, Google Fonts CSS `@import` và Chart.js từ jsDelivr CDN.
- Có 9 test Node trong `test/gacha-math.test.js`.
- Code thực tế đang dùng LocalStorage schema `1.6`, không phải `1.5` như ghi chú bàn giao. Phải giữ schema `1.6`; ngôn ngữ được lưu bằng key độc lập và không kích hoạt reset/migration dữ liệu gacha.
- `index.html` chứa nhiều inline style và chuỗi tiếng Việt; `js/app.js` và `js/chart-helper.js` cũng chứa chuỗi động cần dịch.

## 2. Kiến trúc và thay đổi triển khai

### 2.1. Build và package offline

- Nâng `package.json` lên `1.1.0` và thêm dependency runtime `chart.js`. Import `chart.js/auto` từ source để esbuild đưa Chart.js vào `js/bundle.js`; xóa script CDN khỏi HTML.
- Loại bỏ Google Fonts và dùng font stack hệ thống để không phải tải hoặc phân phối font/licence bổ sung. CSS, JavaScript và mọi asset runtime phải là file nội bộ với đường dẫn tương đối.
- Giữ `npm run build` để tạo bundle phát triển có kiểm soát từ `js/app.js`; không tạo source map.
- Thêm script Node `scripts/build-offline.mjs` để xóa và tạo lại `dist/`, chạy build production minified, rồi chỉ sao chép:
  - `index.html`;
  - `css/style.css`;
  - `js/bundle.js`;
  - `docs/gacha_rules.md`, `docs/strategies.md` và `data/bookkeeping.md` vì giao diện liên kết trực tiếp tới các tài liệu này;
  - file metadata build nếu metadata được hiển thị mà không cần `fetch` khi chạy qua `file://`.
- Truyền version, commit SHA ngắn và thời điểm build vào bundle bằng hằng số do esbuild `define`; hiển thị trong footer. Không đọc metadata bằng `fetch`, vì mở trực tiếp `file://` có thể bị chặn CORS.
- Thêm script Node `scripts/package-offline.mjs` dùng thư viện ZIP cross-platform để tạo `release/endfield-gacha-simulator-v1.1.0-offline.zip`. Cấu trúc ZIP phải mở trực tiếp ở thư mục gốc, không yêu cầu đi qua một thư mục lồng không cần thiết.
- npm scripts cuối cùng:
  - `npm run build` — tạo `js/bundle.js` dùng cho source/repository;
  - `npm run build:offline` — tạo mới `dist/` production;
  - `npm run package` — build offline, kiểm tra manifest và tạo ZIP;
  - `npm test` — chạy toàn bộ test hồi quy.
- Thêm `dist/`, `release/*.zip` và thư mục kiểm tra tạm vào `.gitignore`. Tiếp tục commit `js/bundle.js` vì `index.html` trong repository đang chạy trực tiếp từ bundle này. Không commit ZIP hoặc nội dung `dist/`; ZIP chỉ đính kèm GitHub Release.

### 2.2. Hệ thống i18n

- Tạo `js/i18n.js` là điểm duy nhất quản lý ngôn ngữ, gồm:
  - catalog đầy đủ `vi` và `en` có cùng tập key;
  - `t(key, params)` cho nội suy biến;
  - `formatNumber(value, options)` dựa trên `Intl.NumberFormat`, dùng locale `vi-VN` hoặc `en-US`;
  - `getLocale()`, `setLocale(locale)`, `subscribe(listener)` và hàm áp dụng bản dịch cho DOM;
  - fallback về tiếng Việt khi locale/key không hợp lệ, đồng thời báo key thiếu trong môi trường phát triển.
- Mặc định `vi`; lưu lựa chọn tại `a9e_gacha_locale`. Không thêm locale vào object schema `interactive_state`, `simulator_settings` hoặc `last_results`.
- Chuỗi tĩnh trong HTML dùng `data-i18n`, `data-i18n-title`, `data-i18n-aria-label` và key rõ nghĩa. Không giữ nội dung giao diện chỉ bằng tiếng Việt trong HTML, ngoại trừ fallback tối thiểu cần thiết trước khi JavaScript tải.
- Chuyển toàn bộ chuỗi động trong `app.js` sang `t(...)`: banner, nút quay, inventory, trạng thái pity, luck rating, milestone, cảnh báo, loading, tiêu đề bảng và định dạng kết quả.
- Chuyển tên/mô tả chiến thuật sang key i18n gắn với strategy ID; không thay đổi ID hoặc logic trong `strategies.js`.
- `chart-helper.js` nhận translator/locale hoặc import API i18n tập trung để dịch title, dataset, trục và tooltip. Khi đổi ngôn ngữ, hủy/vẽ lại biểu đồ từ `last_results` đang ở bộ nhớ; không chạy lại Monte Carlo.
- Nút `VI / EN` nằm góc trên bên phải, là nhóm điều khiển có trạng thái `aria-pressed`. Đổi ngôn ngữ chỉ render lại text/format, không reload trang và không reset ví, pity, inventory, form, tab đang mở hoặc kết quả mô phỏng.

### 2.3. UI/UX và accessibility

- Tách inline style khỏi `index.html` sang các class có tên theo chức năng. Duy trì toàn bộ ID DOM mà `app.js` đang sử dụng, hoặc cập nhật đồng bộ source và test nếu đổi ID.
- Header gồm nhận diện ứng dụng, hai tab chính, bộ chọn ngôn ngữ và thông tin version. Tab dùng `role="tablist"`, `role="tab"`, `aria-selected`, `aria-controls`; hỗ trợ Enter/Space và phím mũi tên trái/phải.
- Tab quay tương tác ưu tiên theo thứ tự:
  1. banner và pity 6★/80, pity 5★, guarantee 120;
  2. ví tài nguyên gồm vé nhân vật, Arsenal, Bond Quota, free limited, Dossier hiện tại/kế tiếp;
  3. nút hành động và bảng reveal;
  4. thống kê, luck rating và inventory.
- Hiển thị tiến độ pity 80/120 bằng số và progress bar có `aria-valuemin`, `aria-valuemax`, `aria-valuenow`. Dossier và Bond Quota phải có mô tả ngắn/tooltip về vòng đời và mốc đổi 25 Quota thành một vé.
- Tab Monte Carlo đặt control panel trước bảng kết quả, dùng fieldset/legend hoặc label liên kết đúng cho mọi input/toggle. Nút chạy có disabled/loading state và vùng kết quả dùng `aria-live="polite"`.
- Bảng so sánh nằm trong container cuộn ngang, có caption và chỉ dẫn cuộn trên màn hình nhỏ. Header quan trọng được giữ dễ đọc; không loại bỏ cột dữ liệu.
- Tooltip phải hoạt động bằng hover lẫn keyboard focus. Mọi button/link/input có `:focus-visible`; disabled state đủ tương phản; vùng click tối thiểu hợp lý.
- Breakpoint kiểm tra tối thiểu: 360×800, 768×1024, 1280×720 và 1440×900. Mobile dùng một cột, action button không tràn, bảng/biểu đồ không làm body overflow ngang. Desktop giữ control panel dễ quan sát và ưu tiên số liệu quan trọng.
- Tôn trọng `prefers-reduced-motion`; không dùng animation gây cản trở đọc kết quả.

## 3. LocalStorage và tương thích

- Giữ `SCHEMA_VERSION = '1.6'` và nguyên cấu trúc các key:
  - `a9e_gacha_interactive_state`;
  - `a9e_gacha_interactive_inventory`;
  - `a9e_gacha_simulator_settings`;
  - `a9e_gacha_last_results`;
  - `a9e_gacha_active_banner_idx`.
- Thêm duy nhất `a9e_gacha_locale` với giá trị `vi` hoặc `en`.
- Việc refactor render không được thay đổi dữ liệu số, thứ tự quay, RNG hoặc luồng lưu state. Phải có test/kiểm tra trình duyệt xác nhận dữ liệu schema `1.6` có trước khi nâng cấp vẫn được tải nguyên vẹn.
- Không đưa chuỗi đã dịch vào dữ liệu persisted; inventory tiếp tục lưu dữ liệu ngữ nghĩa như rarity/type/note và dịch khi render.

## 4. Kiểm thử và tiêu chí chấp nhận

### 4.1. Test tự động

- Giữ nguyên và chạy thành công 9 test luật hiện tại.
- Thêm test i18n:
  - catalog VI/EN có tập key giống nhau và không có giá trị rỗng;
  - `t()` nội suy đúng, fallback đúng và không trả key thô cho các chuỗi production;
  - định dạng số theo hai locale;
  - locale không hợp lệ quay về `vi`;
  - đổi locale không sửa/xóa các key LocalStorage schema `1.6`.
- Thêm test package/offline:
  - `dist/` chỉ chứa allowlist;
  - không có `http://`, `https://`, `//cdn`, Google Fonts, jsDelivr, `fetch`, source map hoặc đường dẫn tuyệt đối trong file runtime;
  - mọi `script[src]`, `link[href]`, liên kết tài liệu và CSS `url(...)` cục bộ đều trỏ tới file tồn tại;
  - ZIP giải nén có cùng manifest/checksum với `dist/` và không chứa source, test, script phát triển, `node_modules`, `.git` hoặc file map.
- Không sửa expected value của test luật chỉ để làm test qua; mọi thay đổi logic phải có bug report/bằng chứng riêng.

### 4.2. Kiểm tra trình duyệt

- Mở bản giải nén bằng `file://.../index.html`, không dùng server và không dựa vào `node_modules` của workspace.
- Chặn toàn bộ network trước khi tải trang. Trang phải tải đầy đủ, Chart.js hoạt động, không có request thất bại và không có lỗi console.
- Smoke test cả VI và EN:
  - quay free, x1, x10 và weapon Issue;
  - đổi banner, nhận/tiêu Dossier, cập nhật Bond Quota và pity;
  - chạy Monte Carlo, xem card/bảng/biểu đồ/tooltip;
  - đổi ngôn ngữ sau khi đã quay và sau khi có kết quả mô phỏng;
  - reload và xác nhận locale cùng toàn bộ trạng thái được giữ.
- Kiểm tra keyboard: chuyển tab, chọn VI/EN, focus tooltip, nhập form, bật/tắt paid options và chạy mô phỏng không cần chuột.
- Kiểm tra bốn viewport đã nêu, không có nội dung chính bị che/cắt hoặc body overflow ngang ngoài container bảng chủ ý.

## 5. Tài liệu và phát hành

- Cập nhật README song ngữ với hai luồng:
  - người dùng cuối tải ZIP, giải nén và mở `index.html`;
  - nhà phát triển cài dependency, build, test và package.
- Tạo release notes VI/EN cho `v1.1.0`, nêu rõ offline runtime, UI responsive/accessibility, VI/EN và việc giữ nguyên luật/schema `1.6`.
- Quy trình hoàn tất:
  1. `npm ci`;
  2. `npm run build`;
  3. `npm test`;
  4. `npm run build:offline`;
  5. `npm run package`;
  6. giải nén ZIP vào thư mục tạm ngoài repository và chạy bộ kiểm tra offline;
  7. kiểm tra `git status`, chỉ commit source, lockfile, tài liệu và `js/bundle.js`;
  8. commit lên `main`, push `origin/main`;
  9. tạo annotated tag `v1.1.0`, push tag;
  10. tạo GitHub Release từ tag, dán release notes song ngữ, đính kèm ZIP và checksum SHA-256.
- Nếu thiếu xác thực GitHub, vẫn phải hoàn tất source, test, ZIP, checksum, commit và tag cục bộ; báo chính xác lệnh push/release bị chặn, không tuyên bố release đã phát hành.

## 6. Definition of Done

`v1.1.0` chỉ được coi là hoàn tất khi toàn bộ test cũ/mới qua, ZIP mở trực tiếp không cần server hoặc mạng, console không có lỗi, hai ngôn ngữ bao phủ toàn bộ giao diện động/tĩnh, state schema `1.6` được bảo toàn, responsive/keyboard smoke test đạt và GitHub Release có ZIP cùng checksum (hoặc phần GitHub được ghi rõ là blocker xác thực duy nhất).
